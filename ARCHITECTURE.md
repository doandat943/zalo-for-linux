# Architecture

This document explains how Zalo for Linux works under the hood.

## Overview

This project is not a from-scratch rewrite of Zalo. Instead, it:

1. Downloads the official macOS `.dmg` file.
2. Uses `7z` to extract the `app.asar` archive, which contains the main application logic written in JavaScript.
3. Removes incompatible native macOS files.
4. Wraps the extracted application in a minimal, Linux-compatible Electron shell.
5. Uses `electron-builder` to package everything into a single, portable `AppImage` file.

## Directory Structure

```
zalo-for-linux/
├── main.js                  # Electron entry point
├── package.json             # Project metadata + build config
├── scripts/                 # Build/setup scripts
│   ├── main.js              # Orchestrator (runs setup + build)
│   ├── check-versions.js    # Check for newer Zalo versions
│   ├── download-dmg.js      # Download macOS DMG
│   ├── prepare-app.js       # Extract app.asar + apply patches
│   ├── prepare-zadark.js    # Build ZaDark dark-mode plugin
│   ├── build.js             # Package into AppImage
│   └── patches/             # Individual patch scripts
│       ├── patch-titlebar.js
│       ├── patch-linux-startup.js
│       ├── patch-auto-launch.js
│       ├── patch-sqlite3.js
│       ├── patch-pasting-img.js
│       └── patch-db-cross-v4.js
├── plugins/                 # Git submodules
│   ├── zadark/              # Dark mode extension
│   └── zalux/               # Linux UX tweaks
├── nativelibs/              # Linux reimplementations of Zalo native addons
│   ├── builder.js           # CLI build helper
│   └── db-cross-v4/         # Backup decryption addon
├── app/                     # Extracted app (gitignored, regenerated by build)
├── temp/                    # DMG download cache
└── dist/                    # Build output (AppImage, etc.)
```

## Build Pipeline

`npm run main` runs the full pipeline:

1. **check-versions** — Compares local vs latest Zalo release
2. **download-dmg** — Downloads `ZaloSetup-universal-<version>.dmg` to `temp/`
3. **prepare-zadark** — Builds ZaDark assets (cloned from submodule)
4. **prepare-app** — Extracts `app.asar` to `app/` and applies patches
   via `scripts/patches/`:
   - `patch-titlebar.js` — Enable native title bar
   - `patch-sqlite3.js` — Replace macOS sqlite3 with Linux binary
   - `patch-pasting-img.js` — Patch clipboard helpers + paste handler for text/image paste (Wayland + X11)
   - `patch-db-cross-v4.js` — Build db-cross-v4 from source + patch binding.js + spoof macOS platform ID (25 → 23)
5. **build** — Packages everything into `dist/Zalo-<version>.AppImage`

## Extracted App Patches

When the macOS `app.asar` is extracted, certain modifications are needed
for Linux compatibility. These are applied in `scripts/prepare-app.js`:

| Patch | Why |
|-------|-----|
| `patch-titlebar.js` | `T,frame:!1` → `T,frame:!0` — Enable native title bar |
| `patch-linux-startup.js` | Validate required startup files and consistently map Linux to Zalo's compatible macOS desktop client type in main, compact-app, and utility-process bundles |
| `patch-auto-launch.js` | Lazily initialize Zalo's launcher on Linux, use the stable AppImage path, and make auto-launch IPC handlers return safe results instead of dereferencing `undefined` |
| `patch-sqlite3.js` | macOS `node_sqlite3.node` → Linux one — Avoid "invalid ELF header" crash |
| `patch-pasting-img.js` | Replace Zalo's clipboard bridge and paste handler so `Ctrl+V` works for text, raw images, and copied image files. Falls back to `wl-paste` (Wayland) and `xclip` (X11) when Electron's clipboard API returns empty |
| `patch-db-cross-v4.js` | Build db-cross-v4 from source and patch `binding.js` to load the Linux native addon |

For more details on the native addons, see [nativelibs/README.md](./nativelibs/README.md).

## Why an Electron shell?

The extracted Zalo app is an Electron app designed for macOS. We:

1. Provide a minimal `main.js` that:
   - Imports the ZaDark plugin
   - Imports the screenshot plugin
   - Loads `app/main-dist/main.js` from the extracted app
2. Use our own `package.json` with Linux-specific electron-builder config
3. Rename the extracted `package.json` to `package.json.bak` to avoid conflicts

## What lives in the extracted app/?

Everything in `app/` is **regenerated** by the build pipeline and is
gitignored. It contains:

- `main-dist/main.js` — Zalo's main process bundle
- `main-dist/preload-*.js` — Preload scripts
- `main-dist/compact-app.js` — Utility process
- `pc-dist/` — Renderer/UI bundles
- `native/nativelibs/` — Native addons (some replaced with Linux versions)

The extracted code is proprietary to VNG/Zalo. We don't modify it directly
in version control — patches are applied each build via `prepare-app.js`.
